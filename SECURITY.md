# Security Policy

## Reporting
Do not open a public issue for security-sensitive defects.

Use a private report channel with:
- affected command or package
- impact
- reproduction steps
- suggested mitigation if available

If no private channel is set yet, contact the repository owner directly before disclosure.

## Current Focus
Report issues involving:
- approval bypass
- execution without approval
- audit tampering
- state transition bypass
- unsafe fail-open behavior
